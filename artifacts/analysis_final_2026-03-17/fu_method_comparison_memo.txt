BACKGROUND
----------
Dataset B (analysis_postfix_validated.csv.gz) had 72,872 rows but only
46,720 (64.1%) had co2_per_1000kcal. Four candidate methods exist for
computing the functional unit denominator (kcal_delivered).

METHODS
-------
A. Old stored:      From March 16 validated dataset (old run_chunk pipeline).
                    Payload source unknown; only available for 46,720 rows.

B. Exo draw:        payload_lb from sample_exogenous_draws(cfg, seed).
                    Uses cfg$cargo$payload_lb triangular(8k,22k,42k).
                    Deterministic per seed. Applied to 26,152 missing rows.

C. Audit uniform:   payload_kg = payload_max_lb_draw * load_fraction * 0.453592.
                    From tools/audit_analysis.R lines 20-25.
                    Applied uniformly to ALL 72,872 rows.

D. Phase2 valid.:   payload_kg = actual_units_loaded * unit_weight_lb * 0.45359237.
                    Load-model (cube+weight constrained) from validated 80-row run.
                    Requires units_per_case config (missing in production config).

KEY FINDING
-----------
No two methods produce the same FU values. The four methods use different
payload definitions:
  A uses an unknown old-pipeline payload.
  B uses a stochastic cargo-weight draw (mean ~11 tonnes).
  C uses trailer max capacity * load fraction (mean ~19 tonnes).
  D uses cube+weight-constrained product mass (dry ~16t, refrig ~3.4t).

Method C (audit uniform) is the only one that:
  1. Can be applied to ALL 72,872 rows.
  2. Uses only columns present in every row.
  3. Matches the documented analysis-stage derivation.
  4. Produces a single consistent denominator definition.
